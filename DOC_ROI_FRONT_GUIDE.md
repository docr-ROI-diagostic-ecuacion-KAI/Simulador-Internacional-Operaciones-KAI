# DOC ROI front guide aplicado

Este proyecto debe mantenerse como una herramienta DOC ROI/KAI, no como una landing generica.

## Identidad

- Nombre: Simulador Procesos Internacionales Operaciones KAI.
- Tono: profesional, academico, ejecutivo y formativo.
- Metafora: diagnostico, signos vitales, botiquin, pildoras, vitaminas, medicinas y prescripcion.

## Reglas

- Usar Inter con pesos altos en titulos, botones y etiquetas.
- Mantener letter-spacing en 0 para evitar problemas de legibilidad y encaje responsive.
- Mantener header y hero oscuros.
- Usar logo DOC ROI en primer viewport y footer.
- Cards con radio grande, borde suave y sombra fria.
- Botones tipo pastilla.
- No usar paletas doradas, moradas ni gradientes SaaS genericos.
- En movil, una columna, botones a ancho completo y sin solapes.

## UX obligatoria

La pantalla siempre debe explicar:

1. Donde estoy.
2. Que variable introduzco.
3. Para que sirve esa variable.
4. Como se procesa el dato.
5. Que resultado obtengo.
6. Que decision o validacion sigue.
